# DECISIONS

1. Использован PostgreSQL для корректной поддержки конкурентных операций.
2. Защита от race condition реализована через атомарный conditional UPDATE.
3. Статусы реализованы через enum Prisma.
4. Использован сервисный слой для отделения бизнес-логики от контроллеров.
5. Docker Compose выбран для упрощённого локального запуска.
6. Авторизация упрощена (сессии + сидовые пользователи).
7. Минимум два автотеста: создание заявки и проверка гонки.
